## 1. 무엇을 만들려 했는가 (약 40초)

**대사:** 
"안녕하세요! 이번 프로젝트 발표를 맡은 OOO입니다.

저는 이번에 **LibreFlow**라는 서버리스 도서대여 웹사이트를 만들었습니다. 

쉽게 말해 '온라인 도서관'인데요.

가장 중점적으로 기획했던 부분은 프론트엔드와 백엔드의 깔끔한 분리였습니다.

프론트는 오직 HTML, CSS, 자바스크립트만 사용해서 가볍게 만들었고요,

백엔드와 DB, 인증 시스템은 **Supabase(수파베이스)**라는 서비스를 활용했습니다. 

덕분에 서버 없이도 닉네임 중복 체크, 도서 재고 연동, 연체자 차단, 그리고 비밀번호 찾기 같은 복잡한 기능들이 전부 안전하게 돌아가도록 구성했습니다."

## 2. 프롬프트는 어떤 식으로 작성했는가 (약 1분)

**대사:** 
"사실 이번 프로젝트의 진짜 핵심은 **AI 프롬프트를 어떻게 짰느냐**에 있습니다.

유명한 AI 개발자인 '안드레이 카파시'가 이런 말을 했어요. 

*'AI에게는 맥락과 제약조건을 명확히 줘야 한다'*고요. 

그래서 저는 무작정 코드를 짜달라고 하지 않고, **'프롬프트 블루프린터(청사진)'** 기법을 썼습니다.

먼저 AI한테 '이 프로젝트를 완성하려면 내가 너한테 어떤 순서로 프롬프트를 던져야 할까? 네가 질문 리스트를 짜봐'라고 시킨 거죠.

그러니까 AI가 똑똑하게 '1단계는 인증, 2단계는 DB 세팅, 3단계는 대여 로직' 

이렇게 구조를 잡아주고, 각 단계별로 제가 입력해야 할 완벽한 프롬프트를 만들어주더라고요. 

저는 그걸 바탕으로 뼈대를 잡고 디테일을 채워나갔습니다. 

결국 **구체적으로 지시해야 완벽한 코드가 나온다**는 걸 배웠습니다."

## 3. 웹 시연 및 사용된 프롬프트 (약 2분 30초)

**대사:** 
"백문이 불여일견이니, 제가 만든 웹사이트를 직접 시연하면서 말씀드릴게요."

<br>

**행동:** 메인 화면 띄우기

**대사:** 
"우선 메인 화면입니다. '시작하기'를 눌러 테스트 계정인 tester01로 로그인해 보겠습니다."

<br>

**행동:** '시작하기' 클릭 -> `tester01` / `password123` 로그인 진행 -> 대시보드 진입

**대사:** 
"로그인하면 제 개인 대시보드로 들어옵니다. 바로 도서를 하나 검색해서 빌려보겠습니다."

<br>

**행동:** 도서 검색창에 '해리' 입력 -> '대여하기' 클릭 -> 화면 하단의 '내 대여 현황' 표 보여주기

**대사:** 
"대여를 누르면 오른쪽 아래 알림이 뜨고, 하단에 방금 빌린 책이 추가됩니다. 책 재고도 DB와 연동돼서 실시간으로 깎이게 됩니다."

<br>

**행동:** 새 브라우저 탭(또는 시크릿 창)을 열어 미리 로그인해둔 `tester02` 화면을 보여줌

**대사:** 
"이게 진짜 서버리스 통신이 되는지 확인하기 위해, 제가 옆 창에 `tester02`라는 다른 유저로도 띄워놨는데요.

보시다시피 방금 tester01이 빌린 책의 재고가 tester02의 화면에서도 새로고침 없이 실시간으로 줄어든 걸 볼 수 있습니다."

<br>

**행동:** tester01 화면으로 돌아와 우측 상단 '로그아웃' 버튼 클릭 후, `admin` / `password123` 계정으로 다시 로그인 

**대사:** 
"이번엔 로그아웃하고, 시스템의 핵심인 **관리자(admin)** 계정으로 들어가 보겠습니다."

<br>

**행동:** 로그인 완료 후, 대시보드 하단에 새롭게 나타난 '관리자 전용 패널'을 마우스로 가리키며 스크롤

**대사:** 
"관리자로 로그인하면 DB 보안 정책(RLS) 덕분에 일반 유저에겐 안 보이던 '관리자 패널'이 열립니다. 

여기서는 누가 어떤 책을 빌려갔는지 전체 현황을 다 볼 수 있고, 연체된 책을 강제로 반납시킬 수도 있습니다."

<br>

**행동:** 다시 로그아웃 후 로그인 화면 하단 '비밀번호 재설정 가이드' 클릭 -> 모달 창에서 'tester01' 입력 후 조회

**대사:** 
"마지막으로 아이디를 까먹었을 땐, 아이디만 치면 마스킹된 형태로 이메일을 찾아주고 리셋 링크를 보내주는 보안 기능도 갖췄습니다."

<br>

**대사:** 
"이 모든 기능들을 만들 때, 저는 이런 식의 프롬프트를 썼습니다.

*(발표 화면에 프롬프트 띄우거나 손으로 가리키며)*

'Supabase Auth로 로그인 만들고, 프로필 테이블에 RLS 보안 걸어줘' 라든지, 

'대여/반납은 RPC 함수로 트랜잭션 처리해줘' 처럼 

개발 용어를 섞어서 아주 명확하게 지시했습니다."

## 4. 느낀 점 (약 1분)

**대사:** 
"마지막으로 느낀 점입니다. 

첫째, **AI는 압도적인 생산성 도구**였습니다. 

백엔드 지식이 얕아도 프롬프트만 잘 설계하면 혼자서 풀스택 웹앱을 며칠 만에 뚝딱 만들 수 있었습니다.

둘째, 하지만 **AI는 마법이 아니었습니다.** 

코드를 잘 짜주긴 하는데, 자세히 들여다보면 DB 권한 설정(RLS)을 뚫리게 해 놓거나 에러 처리를 대충 넘기는 경우가 꽤 많았습니다. 

결국 개발자가 코드를 읽고 논리를 검증하지 못하면, 겉보기에만 그럴싸하고 속은 텅 빈 프로그램이 나오겠구나 하는 걸 뼈저리게 느꼈습니다.

결론적으로 AI는 '모든 걸 다 해주는 요술램프'가 아니라, 

**'내가 똑바로 지시하고 검증해야만 일 잘하는 조수'**라는 것을 깨달았습니다. 

이상으로 발표를 마치겠습니다. 감사합니다!"
